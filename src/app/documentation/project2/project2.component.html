Requirements & Interview Findings

Requirements

For the requirements we decided that we would have four pages, one for the main page and one for each of the classes. On the main page we wanted the user to be able to see and interact with all of the most important info very easily and cleanly, Assignments, Announcements, and Chat for that class.

However on the class pages we wanted there to be tabs that would display important fields one after another that contained the more detailed relevant data about each topic. Within the class pages we wanted to have the chat boxes be accessible in the bottom right of the page. These chats only access data for that class.

Interview Findings

  • What is your primary method of communicating of your teacher outside of in-person?
    • Email or discord
    • Why do you use this method?
      • Discord is faster, email is reliable if they don't have discord
      • Most professors reply to their email within a day or so
      • Professors provide their email as their primary contact
  • What is your primary method of communicating with classmates outside of in-person?
    • Discord, teams, zoom, or group chat
      • Why do you use this method?
        • Group chat is fast, the other methods are good for working on the assignments together, and include screen share options
        • People are on Discord when I need to talk to them so it is just faster than trying to contact them some other way
        • I use discord for purposes other than school so having using it for school is convenient.
  • To what extent do students ask questions to teachers directly about assignments?
    • When the assignment is first assigned, students ask questions to the professor. After that, only when the rest of the students can't answer the questions does the student ask the professor.
    • I ask questions when I'm stuck when a question is confusing or if the homework is taking too long and I might be able to get an extension
  • To what extent do students ask questions to teachers and/or other students about assignments via online communication platforms like Discord?
    • Only ask teachers on big problems as a last resort
    • Ask students for any sort of problem
    • If my professor is on discord I will talk to them sometimes most of the time I'd prefer in person or e-mail
  • How often do you ask questions?
    • Not often for teachers, often for students
    • I ask questions often when trying to learn a new subject
    • Whenever I am not able to find the answer on my own via the class materials, textbook or internet. And then I almost always go to other students first.
  • Would you be more likely to ask a professor questions if you could easily talk to them?
    • Only if other students don't know the answer
    • Yes
    • As long as they were an approachable professor.

Contextual Inquiry Findings

In one contextual inquiry the participant heavily used the Calendar view to track their assignments, this was something that surprised me because I don't normally use the calendar view and neither do most of the people who I had preliminary talks with. My main takeaway was that the user wanted to be able to view all of their assignments and relevant materials and know when they were due or scheduled for.

Detailed Description of Interface

Main Page

Main page and Header devleoped by Nishanth Chidambaram. Chat feature developed by Josh Zack

The main starting page that you will first see. It currently shows the three main classes, UI, Senior Design, and Computer Graphics.

For each of the three classes we show an accordion with the Assignments, Announcements, and Chat content for said classes.

The accordion works by letting the content of the 3 sections collapse into a title box, and a maximum of 1 box can be expanded in each accordian. This allows the information for each class to be viewed and navigated in a more user-friendly way

The chat accordion section has full chat functionality, letting the user input a file or text to be able to chat with the class.

Class Page

Class page and individual tabs developed by Ethan Fidler. Chat feature developed by Josh Zack

The class page allows a student to access all of the information regarding their class.

The BACK TO DASHBOARD button returns the user to the main page.

The CHAT button at the bottom right and corner of the screen opens the page's chat window, which pops out and can be moved freely throughout the window.

The Tabs on the left side of the page allow the user to select which information they wish to view. The availible Tabs are the Assignments tab, the Announcements tab, the Grades tab, the Course Materials tab, the Syllabus tab and the Zoom tab. The Assignments tab has a badge which displays how many announcements have been posted. The Zoom tab has not been implemented.

Assignments Tab

The Assignments tab is a list of all assignments that have been posted in a given class. The are sorted by due date and indicate via color whether they are past due, or close to their due date or not. When the user clicks on an assignment it opens the accordion revealing additional information about the assignment as well as a VIEW ASSIGNMENT button which takes the user to the Assignment's Item Page which will be covered in the Item Page section.

Announcements Tab

The Announcements tab has a list of all announcements sorted by post date. Clicking an announcement reveals the content of the announcement.

Grades Tab

The Grades tab lists all graded assignments in a table. The table lists the grade type, due date, current grade (or "--" if ungraded) and the percentage. Each row also has a link to the item's page.

At the bottom the total grades can be calculated from the grade type weights and their total contributions.

Course Materials Tab

The Course Materials tab lists all of the materials associated with the course in a table. They are sorted by Module and post Date. Additionally there is a VIEW PAGE button that allows the user to view each item's page.

Syllabus Tab

The Syllabus tab displays the course's syllabus to be scrolled though.

At the bottom of the page there is a list of action buttons to be taken with the Syllabus. The DOWNLOAD button downloads the syllabus as an html file. The ASK A QUESTION allows the user to ask questions in a forum style. The VIEW QUESTIONS button allows the user to view and reply to other students' questions and see the answers to them. The questions dialogs will be shown and detailed in the Item Page section.

Chat Popout



When the CHAT button is clicked in any of the class pages this window will appear. The user can then see any messages sent by others, (hardcoded for demo) or send messages/files of their own. The window can be moved around by clicking the red header bar with the class name. Additionally the user can change the size of the window by dragging the three diagonal lines in the bottom right.

There is a gray bar at the bottom for sending files, or sending messages. The file icon, left button, will open a popup window which will let the user select a file to be sent.

Next to the file there is a text field that you can type messages of any size in. If you hit Enter or the button to the right of this field then the message will be sent and shown in the message window above.

Lastly if you click the white X in the top right of the window then you can close the chat window. Reclicking the CHAT button again, the window will reappear in the same spot as it was closed.

Item Page

Item page and questions funtionality developed by Ethan Fidler

Each Item listed in Assignments or Course Materials has it's own item page where it's information can be viewed inlcuding post date, due date, points and it's submission status.

At the bottom there is a list of buttons. The DOWNLOAD button downloads the respective file.

If the item filetype is 'html' The PREVIEW ITEM button shows the modal above which contains the file that the user will be downloading/watching.

If the item requires the user to submit something then the SUBMIT button allows the user to upload a file submission or optionally add text entry. This updates the submission status from the main page.

the VIEW QUESTIONS button opens the dialog above that displays all questions and replies left by other students and instructors. The user can then click ASK A QUESTION to open the following modal.

The above modal appears when the user clicks ASK A QUESTION. The user can type in a text entry and choose to post the question anonymously. The SUBMIT button posts the question.
